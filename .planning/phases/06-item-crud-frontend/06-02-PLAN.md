POST /api/items
  Request: { "name": "string", "description": "string|null", "container_id": int, "tags": ["string"] }
  Response 201: ItemResponse { id, container_id, container_label, name, description, tags: [], created_at, updated_at }
  Errors: 400 "name is required", 400 "at least one tag is required", 404 "container not found"

GET /api/items/{itemID}
  Response 200: ItemResponse
  Errors: 404 "item not found"

PUT /api/items/{itemID}
  Request: { "name": "string", "description": "string|null", "container_id": int }
  Response 200: ItemResponse (NOTE: PUT does NOT touch tags -- use tag endpoints)
  Errors: 400, 404

DELETE /api/items/{itemID}
  Response: 204 No Content
  Errors: 404

POST /api/items/{itemID}/tags
  Request: { "name": "string" }
  Response 200: ItemResponse (full item with updated tags)
  Errors: 400, 404

DELETE /api/items/{itemID}/tags/{tagName}
  Response: 204 No Content
  Errors: 404

GET /api/containers/{containerID}/items
  Response 200: ContainerWithItems { id, shelf_id, col, row, label, items: [ItemResponse], created_at, updated_at }
  Errors: 404

<task type="auto">
  <name>Task 1: Complete grid.js rewrite -- inline expansion, CRUD forms, cell updates</name>
  <files>static/js/grid.js</files>
  <action>
Replace the entire contents of static/js/grid.js with a complete rewrite. The file should be a single IIFE (`(function() { 'use strict'; ... })();`).

**Section 1: State and DOM references**
- `gridContainer = document.querySelector('.grid-container')`
- `expandedCell = null` -- tracks currently expanded cell (only one at a time)

**Section 2: Utility functions**
- `formatCount(n)` -- returns `'--'` for 0 (use `\u2014` em-dash), `'1 item'` for 1, `'N items'` for N>1 (per D-18)
- `pulseCell(cell)` -- adds `.pulse` class, removes after 500ms via `animationend` listener (per D-22)
- `updateCellCount(containerId, delta)` -- finds cell by `[data-container-id="X"]`, updates `data-count`, updates `.cell-count` text via `formatCount()`, toggles `.cell-empty` class (per D-17, D-19, D-20)
- `async apiCall(url, options)` -- wrapper around fetch() that returns `{ ok, status, data }`. On network error returns `{ ok: false, status: 0, data: { error: 'Connection error. Please try again.' } }`. Parses JSON response body. For 204 responses, returns `{ ok: true, status: 204, data: null }`.

**Section 3: Expand/Collapse**
- Event delegation on `gridContainer` click. If click target is inside `.grid-cell` and cell is NOT already expanded: call `expandCell(cell)`.
- `expandCell(cell)`:
  1. If another cell is expanded, collapse it first via `collapseCell()`
  2. Save cell's original innerHTML to `cell._originalHTML`
  3. Add `.expanded` class to cell
  4. Clear cell innerHTML
  5. Create close button: `<button class="expanded-close" type="button">X</button>` (use multiplication sign `\u00D7`)
  6. Create header div with class `expanded-header`, text = cell's `data-coord` attribute
  7. Create content div (id not needed, just a reference)
  8. Append close button, header, content div to cell
  9. Fetch items via `GET /api/containers/${cell.dataset.containerId}/items`
  10. If items array is empty or missing: call `renderAddForm(cell, contentDiv, parseInt(cell.dataset.containerId))`
  11. If items exist: call `renderItemList(cell, contentDiv, items, parseInt(cell.dataset.containerId))`
  12. Close button click -> `collapseCell()`
  13. Set `expandedCell = cell`

- `collapseCell()`:
  1. If `expandedCell` is null, return
  2. Remove `.expanded` class
  3. Restore `expandedCell.innerHTML` from `expandedCell._originalHTML`
  4. Set `expandedCell = null`

- Click-outside handler: `document.addEventListener('click', (e) => { if (expandedCell && !expandedCell.contains(e.target)) collapseCell() })`
- Escape key handler: `document.addEventListener('keydown', (e) => { if (e.key === 'Escape') collapseCell() })`

**Section 4: Render item list** (per D-03, D-23)
- `renderItemList(cell, container, items, containerId)`:
  1. Clear container
  2. Create `<ul class="expanded-items">`
  3. For each item, create `<li>` with:
     - `<div class="item-row">`: `<span class="item-name">${item.name}</span>` + tags as `<kbd class="tag-chip">${tag}</kbd>` (no X button in list view -- X only in edit mode) + `<div class="item-actions">` with Edit and Delete buttons
     - Edit button: `<button class="outline secondary" type="button">Edit</button>` -- on click calls `renderInlineEdit(li, item, cell, container, containerId)`
     - Delete button: `<button class="outline" type="button">Delete</button>` -- on click calls `handleDelete(btn, item, cell, container, containerId)`
  4. After the list, add an "Add item" button: `<button type="button">Add item</button>` -- on click calls `renderAddForm(cell, container, containerId)`
  5. Append ul and button to container

**Section 5: Add item form** (per D-02, D-07, D-08, D-09, D-10, D-16)
- `renderAddForm(cell, container, containerId)`:
  1. Clear container
  2. Show "No items" heading + "Add the first item to this container." text if coming from empty cell (check cell `data-count` == 0)
  3. Create `<form class="expanded-form">` with:
     - Name field: `<label>Name<input type="text" name="name" required></label>`
     - Description field: `<label>Description<textarea name="description"></textarea></label>` (per D-10, optional)
     - Tags section:
       a. `<label>Tags</label>`
       b. `<div class="tag-chips-container">` -- holds tag chips
       c. `<input type="text" placeholder="Type tag, press Enter">` -- the tag input
       d. `<small class="tag-hint" hidden>Add at least one tag</small>` -- validation hint
     - Error area: `<small class="form-error" aria-live="polite"></small>`
     - Button row `<div class="form-actions">`:
       a. Submit button: `<button type="submit" aria-disabled="true" disabled>Add item</button>` -- starts disabled
       b. If navigated from item list (not empty cell): Cancel button `<button type="button" class="secondary">Discard changes</button>` that re-fetches and re-renders item list
  4. **Tag input behavior** (per D-08, D-16):
     - Maintain a `pendingTags = []` array in closure
     - On `keydown` of tag input: if key is `Enter`, prevent default (CRITICAL -- prevents form submit per D-16), trim input value, lowercase, check non-empty and not duplicate, add to pendingTags, re-render chips, clear input, call `updateSubmitState()`
     - `renderTagChips(chipsContainer, pendingTags)`: clear container, for each tag create `<kbd class="tag-chip">${tag}<button type="button">\u00D7</button></kbd>`, X click removes from pendingTags, re-renders, calls `updateSubmitState()`
  5. **Submit disabled logic** (per D-09):
     - `updateSubmitState()`: enable submit button only if name input has non-empty trimmed value AND pendingTags.length > 0. Use both `disabled` property and `aria-disabled` attribute.
     - Also listen to `input` event on name field to call `updateSubmitState()`
  6. **Form submit** (per D-12):
     - `e.preventDefault()`
     - Validate name (trim, non-empty -- set `aria-invalid` + "Name is required" if empty)
     - Validate pendingTags.length >= 1 (show tag-hint if 0)
     - Build JSON body: `{ name, description: desc || null, container_id: containerId, tags: pendingTags }`
     - POST to `/api/items` via `apiCall()`
     - On success: `collapseCell()`, `updateCellCount(containerId, +1)`, `pulseCell(cell)` (find cell again by containerId since collapse restores HTML)
     - On error: show error message in `.form-error`

**Section 6: Inline edit** (per D-25, D-07, D-14, D-15)
- `renderInlineEdit(li, item, cell, container, containerId)`:
  1. Save `li._originalHTML = li.innerHTML`
  2. Clear li innerHTML
  3. Create inline form (NOT a `<form>` element to avoid nested forms -- use a div with class `expanded-form`):
     - Name input pre-filled with `item.name`
     - Description textarea pre-filled with `item.description || ''`
     - Tag chips showing `item.tags` with X to remove (each X calls `DELETE /api/items/${item.id}/tags/${tagName}` immediately per D-14)
     - Tag input field to add new tags (Enter adds, calls `POST /api/items/${item.id}/tags` immediately per D-15)
     - "Save" button (accent) + "Discard changes" button (secondary)
  4. **Tag management in edit mode** is LIVE (per D-14, D-15):
     - Removing a tag chip: immediately calls `DELETE /api/items/${item.id}/tags/${encodeURIComponent(tagName)}`. On success, remove chip from DOM and update local tags array. On error, show error message.
     - Adding a tag (Enter in input): immediately calls `POST /api/items/${item.id}/tags` with `{ name: tagValue }`. On success, add chip to DOM and update local tags array. On error, show error.
     - This differs from add mode where tags are batched in the POST body.
  5. **Save button**:
     - Calls `PUT /api/items/${item.id}` with `{ name, description, container_id: containerId }`
     - On success: re-fetch container items, re-render item list. If container_id changed (move): `updateCellCount(oldContainerId, -1)`, `updateCellCount(newContainerId, +1)`, pulse both cells.
     - On error: show inline error
  6. **Discard changes button**: restore `li.innerHTML = li._originalHTML` (note: tags added/removed are already committed to API -- "discard" only discards unsaved name/description changes. This is acceptable since tag ops are immediate.)

**Section 7: Delete with inline confirmation** (per D-24)
- `handleDelete(btn, item, cell, container, containerId)`:
  1. If `btn.dataset.confirm === 'true'`: execute delete
     - Call `DELETE /api/items/${item.id}` via `apiCall()`
     - On success: `collapseCell()`, `updateCellCount(containerId, -1)`, `pulseCell(findCell(containerId))`
     - On error: show alert with error message
  2. Else (first click):
     - Set `btn.dataset.confirm = 'true'`
     - Change text to "Confirm?"
     - Add `.btn-confirm` class (red background from CSS)
     - Set 3-second timeout: revert to "Delete" text, remove `.btn-confirm`, clear `dataset.confirm`
     - Store timeout ID on btn for cleanup

**Section 8: Helper -- find cell by container ID**
- `findCell(containerId)` -- returns `document.querySelector('.grid-cell[data-container-id="' + containerId + '"]')`

**Important implementation notes:**
- All text in English (per D-27). No Polish strings anywhere.
- Use `const` and `let`, arrow functions are fine (modern browsers only).
- The IIFE pattern prevents global scope pollution.
- Event delegation on gridContainer for cell clicks. Expanded cell clicks must NOT re-trigger expand (check if cell has `.expanded` class and return early from grid click handler).
- When cell is expanded and user clicks inside it (on form fields, buttons, etc.), clicks must NOT bubble to the click-outside handler. The click-outside handler checks `expandedCell.contains(e.target)`.
- `pulseCell()` must work after collapse (cell HTML is restored). Find cell by containerId, then add pulse class.
