**JS (grid.js):** Add a reusable `createAutocomplete(tagInput, onSelect)` function inside the IIFE, BEFORE `renderAddForm` and `renderInlineEdit`. This function:

1. Creates a wrapper div with class `autocomplete-wrapper`
2. Wraps the tagInput's current position in the DOM: `tagInput.parentNode.insertBefore(wrapper, tagInput); wrapper.appendChild(tagInput);`
3. Creates a `ul` element with class `autocomplete-dropdown`, appends it to wrapper after tagInput
4. Tracks state: `activeIndex = -1`, `suggestions = []`, `debounceTimer = null`
5. On tagInput `input` event (NOT replacing the existing keydown — this is a NEW listener on `input`):
   - Clear debounceTimer, set new one at 200ms (Claude's discretion on timing)
   - In the debounce callback: get `val = tagInput.value.trim().toLowerCase()`
   - If val is empty, clear dropdown (`ul.innerHTML = ''`) and return
   - Call `apiCall('/api/tags?q=' + encodeURIComponent(val))`
   - If result.ok and result.data is array: filter out tags already present (the `onSelect` callback's parent scope tracks this — see wiring below), take first 5 (per D-02), store in `suggestions`, render as `li` elements
   - Each `li` gets a click listener that calls `onSelect(tagText)` and clears dropdown
   - Reset `activeIndex = -1`
6. On tagInput `keydown` — add a NEW listener that runs BEFORE the existing Enter handler. Use the capture pattern: add listener with `{ capture: true }` or add it first:
   - `ArrowDown`: If dropdown has items, increment activeIndex (wrap to 0), add `.active` class to current li, remove from others. `e.preventDefault()` to prevent cursor movement.
   - `ArrowUp`: Decrement activeIndex (wrap to last), update `.active`. `e.preventDefault()`.
   - `Enter`: If `activeIndex >= 0` and suggestions exist, call `onSelect(suggestions[activeIndex])`, clear dropdown, `e.preventDefault()` AND `e.stopImmediatePropagation()` to prevent the existing Enter handler from also firing. Per D-04, if no suggestion is highlighted (activeIndex === -1), let the event propagate to existing handler (typed text becomes tag as before). Per D-05, even if typed text matches a suggestion, if nothing is highlighted via arrows, let existing handler add it directly.
   - `Escape`: Clear dropdown, reset activeIndex.
7. On tagInput `blur`: setTimeout 150ms then clear dropdown (delay allows click on suggestion to register first).
8. On tagInput `focus`: If tagInput.value.trim() is not empty, trigger the fetch (per D-01, show on focus + any input).

The `createAutocomplete` function returns an object `{ destroy: function() }` that removes all listeners (for cleanup if needed).

**CRITICAL:** The `onSelect` callback signature is `onSelect(tagName)`. The caller provides it. The autocomplete component does NOT manage chip state — it just notifies the caller that a tag was selected, and clears the input + dropdown.

**IMPORTANT ordering:** The createAutocomplete keydown listener (added with capture or added first) MUST intercept Enter BEFORE the existing keydown handler when a suggestion is highlighted. When activeIndex >= 0, it calls `e.stopImmediatePropagation()` which prevents the existing handler from also adding the tag. When activeIndex === -1 (no highlight), it does nothing and lets the existing handler proceed normally.
