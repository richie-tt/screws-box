    **2. JavaScript (grid.js) — add inside the existing IIFE, after the existing cell/panel code:**

    Add a `openSettingsPanel()` function that:
    - Calls `collapseCell()` first to close any open cell panel (per pitfall 4)
    - Creates a fixed panel div with class `settings-panel` (reuses expanded-panel positioning approach)
    - Reads current values from the gear button's data attributes: `data-shelf-name`, `data-shelf-rows`, `data-shelf-cols`
    - Panel HTML structure (per UI-SPEC section 5b):
      ```
      <div class="settings-panel" role="dialog" aria-label="Grid settings">
        <div class="panel-header">
          <span class="panel-coord">Settings</span>
          <button class="expanded-close" aria-label="Close">&times;</button>
        </div>
        <div class="panel-body">
          <label for="settings-name">Shelf name</label>
          <input type="text" id="settings-name" value="{current name}" maxlength="100">

          <div class="settings-grid-inputs">
            <div>
              <label for="settings-cols">Columns</label>
              <input type="number" id="settings-cols" min="1" max="30" step="1" value="{current cols}">
            </div>
            <div>
              <label for="settings-rows">Rows</label>
              <input type="number" id="settings-rows" min="1" max="26" step="1" value="{current rows}">
            </div>
          </div>

          <div class="form-error" id="settings-error" hidden></div>

          <div class="form-actions">
            <button type="button" class="secondary" id="settings-cancel">Discard Changes</button>
            <button type="button" id="settings-save">Save Settings</button>
          </div>
        </div>
      </div>
      ```
    - Appends panel to `document.body`
    - Adds `active` class to the gear button
    - Positions panel near the gear icon (use getBoundingClientRect on the gear button, set top/left on the panel similar to expandCell positioning)
    - For mobile (window.innerWidth <= 860): center panel horizontally with 16px margins

    Close behavior:
    - Close button (X), Cancel button, Escape key, click outside panel all close the panel and remove `active` class from gear button
    - Closing discards any changes (just removes the panel DOM)
    - Return focus to gear button on close

    Save behavior:
    - Read values from inputs
    - Client-side validation: rows must be 1-26, cols must be 1-30. If invalid, show error in `.form-error` element: "1-26 rows allowed (A-Z)" or "1-30 columns allowed". Set `aria-invalid="true"` on the offending input.
    - Set `aria-busy="true"` and `opacity: 0.7` on save button, disable both buttons
    - `fetch('/api/shelf/resize', { method: 'PUT', headers: {'Content-Type': 'application/json'}, body: JSON.stringify({rows: parseInt(rows), cols: parseInt(cols), name: name || undefined}) })`
    - On 200 response: `window.location.reload()`
    - On 409 response: Close settings panel, call `showResizeBlockedModal(data.affected)` (see below)
    - On 400 response: Show error message from response in `.form-error`, re-enable buttons
    - On other errors: Show "An error occurred" in `.form-error`, re-enable buttons

    Add a `showResizeBlockedModal(affected)` function that:
    - Creates overlay div with class `resize-modal-backdrop` (per UI-SPEC section 5c)
    - Creates modal div with class `resize-modal`
    - Modal HTML structure:
      ```
      <div class="resize-modal-backdrop">
        <div class="resize-modal" role="alertdialog" aria-labelledby="resize-modal-title">
          <div class="resize-modal-header">
            <h3 id="resize-modal-title">Cannot Resize</h3>
          </div>
          <div class="resize-modal-body">
            <p>The following containers have items. Move or delete these items before resizing.</p>
            <ul class="resize-blocked-list">
              {for each affected container:}
              <li>
                <span class="position-badge">{label}</span>
                <span class="blocked-item-count">{item_count} item(s)</span>
                <ul class="blocked-item-names">
                  {for each item name: <li>{name}</li>}
                </ul>
              </li>
            </ul>
          </div>
          <div class="resize-modal-footer">
            <button id="resize-modal-ok">Back to Grid</button>
          </div>
        </div>
      </div>
      ```
    - Appends to `document.body`
    - Focus the "Back to Grid" button immediately
    - Close on: clicking "Back to Grid", pressing Escape, clicking backdrop outside modal
    - On close: remove overlay from DOM, return focus to gear button

    Wire the gear button click handler:
    - `document.querySelector('.settings-trigger').addEventListener('click', ...)` — toggle open/close of settings panel
    - If panel is already open, close it. If closed, open it.

    **Mutual exclusion with cell panel:** Call `collapseCell()` before opening settings. Also, in the existing `expandCell()` function (or its equivalent), close settings panel if open before expanding a cell.
