   ```go
   package session

   import (
       "context"
       "encoding/json"
       "fmt"
       "strings"
       "time"

       "github.com/redis/go-redis/v9"
   )

   var _ Store = (*RedisStore)(nil)

   type RedisStore struct {
       client *redis.Client
       ttl    time.Duration
   }

   func NewRedisStore(redisURL string, ttl time.Duration) (*RedisStore, error) {
       opts, err := redis.ParseURL(redisURL)
       if err != nil {
           return nil, fmt.Errorf("invalid REDIS_URL: %w", err)
       }
       client := redis.NewClient(opts)
       // D-06: PING on startup -- fail fast if unreachable
       if err := client.Ping(context.Background()).Err(); err != nil {
           client.Close()
           return nil, fmt.Errorf("redis unreachable: %w", err)
       }
       return &RedisStore{client: client, ttl: ttl}, nil
   }
   ```

   Methods:
   - `Create`: `json.Marshal(sess)` then `client.Set(ctx, "session:"+sess.ID, data, r.ttl)` per D-08
   - `Get`: `client.Get(ctx, "session:"+id)`, unmarshal, return nil,nil if `redis.Nil` error
   - `Delete`: `client.Del(ctx, "session:"+id)` per D-02
   - `Touch`: Get session JSON, unmarshal, update LastActivity to `time.Now()`, Set back with `r.ttl` (refreshes EXPIRE). Handle `redis.Nil` gracefully (return nil -- session already gone).
   - `DeleteByAuthMethod`: SCAN `session:*`, for each key GET+unmarshal, if AuthMethod matches then DEL. Return count. Handle expired keys gracefully per Pitfall 2.
   - `List`: SCAN `session:*` with count=100, GET each key, unmarshal, collect. Skip `redis.Nil` errors (Pitfall 2). Return empty slice (not nil) if no sessions.
   - `Close`: `return r.client.Close()`

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| App -> Redis | Session data crosses network to Redis server |
| Redis -> App | Untrusted data read back from Redis (could be tampered if Redis is compromised) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation | Task |
|-----------|----------|-----------|-------------|------------|------|
| T-16-01 | Information Disclosure | RedisStore | mitigate | REDIS_URL supports rediss:// for TLS. Session IDs are 256-bit random (no guessing). CSRFToken stored in Redis is HttpOnly cookie-bound. No PII beyond username. | Task 2 |
| T-16-02 | Tampering | RedisStore deserialization | accept | Redis is a trusted internal service on home network. JSON unmarshal into typed struct limits impact. If Redis is compromised, attacker already has network access. | Task 2 |
| T-16-03 | Denial of Service | Redis connection failure | mitigate | D-06: fail fast on startup if Redis unreachable. D-07: runtime Redis failure returns errors (no silent degradation). App logs errors for diagnosis. | Task 2, Task 3 |
</threat_model>
