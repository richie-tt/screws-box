## Component Inventory

### 1. Sessions Nav Item (sidebar)

Replace the existing disabled `<span>` with an active `<a>` link:

```
Sessions (N)
```

- Element: `<a href="#sessions" class="admin-nav-item">Sessions <small class="nav-badge">N</small></a>`
- Badge shows count of active sessions (D-13)
- Badge styling: `font-size: 12px; color: var(--text-muted); font-weight: 400`
- Active state uses existing `.admin-nav-item.active` pattern

### 2. Sessions Card (`#sessions`)

Standard `.admin-card` following existing admin pattern.

**Focal point:** The session table is the primary visual anchor -- the user's eye should land on the table rows immediately upon navigating to this section. The card heading and store indicator pill are secondary orientation elements above the table.

**Header row:**
- `<h2>` "Sessions" in DM Mono 1.2rem weight 600
- Right-aligned: store backend indicator pill + Refresh button
- Store indicator: small pill showing "Memory" or "Redis" with `font-size: 12px; padding: 4px 8px; border-radius: 10px; background: var(--bg-inset); color: var(--text-muted)`
- Redis active pill: `background: hsl(145, 45%, 45%, 0.1); color: var(--success)`
- Refresh button: `class="ghost"` with circular arrow SVG icon, 16x16

**Layout:**
```
+--------------------------------------------------+
| Sessions                    [Memory] [Refresh]   |
|--------------------------------------------------|
| User   Method  Created  Last Active  Exp.  Act.  |
| admin  local   10:30    11:45        22h   [Rev] |
| admin  OIDC    09:15    11:40        21h   [---] |  <-- own session
+--------------------------------------------------+
```

### 4. "Your Session" Badge (D-12)

- Element: `<span class="session-badge-own">Your session</span>`
- Styling: `font-size: 12px; font-weight: 600; padding: 4px 8px; border-radius: 10px; background: var(--accent-subtle); color: var(--accent)`
- Replaces the Revoke button in the Actions column for the admin's own session row

### 5. Revoke Button (per row)

- Element: `<button class="ghost danger-ghost">Revoke</button>`
- Styling: `font-size: 12px; color: var(--danger); padding: 4px 8px`
- Hover: `background: hsl(0, 65%, 50%, 0.08)`
- Hidden/disabled on admin's own session row (D-18)

### 6. Revoke All Others Button (D-17)

- Element: `<button class="btn danger" id="revoke-all-btn">Revoke All Others</button>`
- Placement: below the session table, left-aligned in `.admin-form-actions`
- Only visible when there are 2+ active sessions (at least one besides admin's own)

### 7. Revoke Confirmation Dialog (D-16)

Reuse existing `.resize-modal-overlay` / `.resize-modal-dialog` pattern:

- Overlay: `position: fixed; inset: 0; background: rgba(0,0,0,0.5); z-index: 100`
- Dialog: `max-width: 480px; background: var(--bg-raised); border-radius: var(--radius-lg); box-shadow: var(--shadow-lg); padding: var(--space-lg)`
- Title: DM Mono, 1.1rem, weight 600, `color: var(--danger)`
- Actions: flex row, gap `var(--space-sm)`, right-aligned

**Single revoke dialog:**
- Title: "Revoke Session"
- Body: "Revoke session for **{username}**? They will be forced to log in again."
- Dismiss: `class="btn ghost"` "Keep Session"
- Confirm: `class="btn danger"` "Revoke Session"

**Bulk revoke dialog:**
- Title: "Revoke All Other Sessions"
- Body: "Revoke **{N}** sessions? All other users will be forced to log in again."
- Dismiss: `class="btn ghost"` "Keep Sessions"
- Confirm: `class="btn danger"` "Revoke All"

### 8. Empty State (D-15)

When no active sessions (edge case -- admin is always logged in, but shown when session list returns empty):

- Container: centered within card, `padding: var(--space-xl) 0; text-align: center`
- Heading: "No active sessions" in `font-size: 15px; font-weight: 600; color: var(--text)`
- Subtext: "Session store: {Memory|Redis}" in `font-size: 13px; color: var(--text-muted); margin-top: var(--space-xs)`

## Interaction States

### Session Table Loading

- On page load: table renders server-side with current session data (static HTML from Go template)
- Refresh button: fetches `GET /api/sessions`, replaces table body content via JS
- During fetch: Refresh button gets `aria-busy="true"` (existing busy pattern -- opacity 0.7, pointer-events none)
- On error: show feedback message next to Refresh using `.admin-form-feedback.error` pattern

### Revoke Flow

1. User clicks "Revoke" on a session row
2. Confirmation dialog appears (modal overlay pattern from resize modal)
3. User clicks "Revoke Session" to confirm or "Keep Session" to dismiss
4. On confirm: `DELETE /api/sessions/{id}` fires
5. On success: row fades out and is removed from table; session count badge updates
6. On error: dialog stays open, error message appears in dialog body
7. "Revoke All Others" follows same pattern with `DELETE /api/sessions` (bulk endpoint)

### Row Removal Animation

- On successful revoke: `opacity: 0; transition: opacity 0.3s ease` then remove from DOM after transition
- Session count badge in sidebar updates immediately after removal

## Responsive Behavior

Follow existing admin responsive pattern from `admin.css`:

- At `max-width: 768px`: admin layout stacks vertically, sidebar becomes horizontal scroll nav
- Session table: add `overflow-x: auto` wrapper for horizontal scroll on narrow screens
- Table minimum width: 560px (prevents column collapse)
- On mobile, "Expires In" column can be hidden via `display: none` at `max-width: 600px`
